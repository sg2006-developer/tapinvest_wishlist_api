3. Endpoints

3.1 GET /api/v1/bond

Returns full bond catalog (no pagination).

Query ParamTypeAllowedDefault behavior
sortBystringisin, bondYield, minInvestment, tenure, ratingInvalid/missing -> isin
sortOrderstringasc, descInvalid/missing -> asc

Sorting rule: primary sort on requested field with NULLS LAST, stable tiebreaker on isin in same direction. For rating, sorting is based on credit scale (AAA to D).

500 INTERNAL_SERVER_ERROR

3.4 GET /api/v1/wishlist/:wishlistId

Returns one wishlist with bonds.

Path ParamTypeRule
wishlistIdstringMust be valid UUID
Query ParamTypeAllowedDefault behavior
sortBystringmanual, addedRecently, color, yield, minInvestment, tenure, ratingInvalid/missing -> manual

Sort precedence always starts with pinned first: isPinned DESC, then chosen secondary sort:

  • manual: position ASC
  • addedRecently: created_at DESC
  • color: color ASC NULLS LAST, position ASC
  • yield: bond_yield DESC NULLS LAST, position ASC
  • minInvestment: min_investment ASC NULLS LAST, position ASC
  • tenure: tenure ASC, position ASC
  • rating: custom order (AAA to D) ASC NULLS LAST, position ASC
200 OK
{
  "data": WishlistDetailsDto
}
400 BAD_REQUEST
{ "code": "BAD_REQUEST", "message": "Invalid UUID." }

{ "code": "NOT_FOUND", "message": "Requested resource was not found." }

3.12 PATCH /api/v1/wishlist/:wishlistId/reorder

Bulk reorder positions by sending full list of bond ISINs in desired order.

Request body
{
  "bondIsins": ["ISIN_A", "ISIN_B", "ISIN_C"]
}

Validation/business rules:

  • bondIsins required and non-empty
  • must contain exactly all bonds currently in the wishlist (count must match)
  • if any ISIN not in wishlist, request fails
  • updates are transactional
  • reorder updates position only; isPinned is not changed
204 No Content
400 BAD_REQUEST
{ "code": "BAD_REQUEST", "message": "bondIsins is required and must be a non-empty array." }
400 BAD_REQUEST
{ "code": "BAD_REQUEST", "message": "bondIsins must contain all N bonds in the wishlist." }
400 BAD_REQUEST
{ "code": "BAD_REQUEST", "message": "One or more bondIsins do not exist in this wishlist." }
404 NOT_FOUND
{ "code": "NOT_FOUND", "message": "Requested resource was not found." }

500Unexpected server/database error

INTERNAL_SERVER_ERROR500Unexpected backend/database failure.
WISHLIST_LIMIT_REACHED422Attempt to create more than 5 wishlists.
WISHLIST_ALREADY_EXISTS409Attempt to create or rename a wishlist to a name that already exists.
BOND_ALREADY_EXISTS409Attempt to add a bond that is already inside that wishlist.
WISHLIST_FULL422Attempt to exceed 10 bonds in a wishlist.

6. Backend Structural Requirements (for replacement implementation)

  • Keep all routes, HTTP methods, path params, query params, JSON field names, and status/error codes exactly the same.
  • Preserve success envelope shape ({ "data": ... }) and empty-body endpoints exactly.
  • Preserve validation messages where frontend depends on exact text.
  • Keep pinned behavior: pinned bonds always listed before unpinned, regardless of selected sort mode.
  • Keep wishlist limits: max 5 wishlists and max 10 bonds per wishlist.

7. Persistence Schema (current DB contract)

Current DB is PostgreSQL. A compatible replacement should preserve equivalent behavior and constraints.

7.1 Table: bonds

isin VARCHAR(12) PRIMARY KEY
bond_name TEXT NOT NULL
rating VARCHAR(20) NULL
bond_yield DECIMAL(6,2) NULL
min_investment BIGINT NULL
payout_frequency VARCHAR(30) NULL
logo_url TEXT NULL
detail_url TEXT NULL
tenure DECIMAL(6,2) NOT NULL
maturity_date DATE NULL

7.2 Table: wishlists

id UUID PRIMARY KEY
name VARCHAR(50) NOT NULL
created_at TIMESTAMPTZ NOT NULL
updated_at TIMESTAMPTZ NOT NULL

7.3 Table: wishlist_bonds

wishlist_id UUID NOT NULL REFERENCES wishlists(id)
bond_isin VARCHAR(12) NOT NULL REFERENCES bonds(isin)
created_at TIMESTAMPTZ NOT NULL
color VARCHAR(7) NULL
position INT NOT NULL DEFAULT 0
is_pinned BOOLEAN NOT NULL DEFAULT FALSE
PRIMARY KEY (wishlist_id, bond_isin)
